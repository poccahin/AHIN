# Trusted Twin Court Design Tokens Readiness

Source visual reference: `ahin-design-tokens-README.md`

The Trusted Twin visual system is integrated as CSS-ready semantics rather than a separate token build pipeline.

## Adopted Semantics

- Rice-paper panel surface for court artifacts.
- Amber responsibility accent for human-reviewed readiness.
- Blue responsibility accent for machine route and replay evidence.
- Green only for safe readonly or readiness-complete states.
- Red reserved for actual anomaly or revocation states, not decoration.
- Hairline borders for institutional density.
- Monospace identifiers for case IDs, hashes, addresses, and readiness flags.
- Serif text only in judgment/certificate passages.

## Production Boundaries

The design language does not imply:

- completed identity verification,
- completed external signatures,
- chain submission,
- live protocol execution,
- treasury funding authority,
- transfer, burn, or signing capability.
